IDE Pascal e Analisador Lexico em Java

Documentacao do codigo-fonte da pasta IDE-PASCAL
Arquivos estudados: Clone.java, Lexer.java e Token.java
Linguagem: Java | Interface: Java Swing | Tema: Analise Lexica de Mini Pascal

Gerado para estudo do funcionamento, metodos e implementacao do projeto.

1. Visao geral do projeto

O projeto implementa uma pequena IDE para escrever codigo inspirado em Pascal e executar uma analise lexica. A analise lexica e a primeira etapa de um compilador: ela le o texto do programa, divide esse texto em partes menores chamadas tokens e classifica cada parte.

A aplicacao possui editor de codigo, numeracao de linhas, menus de arquivo, botoes de acao, destaque de sintaxe, tabela de tokens, tabela de erros, exportacao de resultado e uma funcao opcional de sugestao com IA local usando Ollama.

Ideia central: o usuario escreve codigo Pascal no editor. Ao clicar em analisar, a classe Clone envia o texto para Lexer. O Lexer percorre o texto e cria objetos Token. A interface mostra os tokens e os erros encontrados.

3. Como o sistema funciona

  1. O usuario digita ou abre um arquivo Pascal no editor.
  2. A interface monitora mudancas para atualizar o titulo, status e destaque visual.
  3. Quando a acao acaoAnalisar() e executada, o texto do editor e passado ao construtor de Lexer.
  4. O metodo analisar() do Lexer percorre o codigo e decide qual metodo de reconhecimento chamar.
  5. Os tokens criados sao armazenados em uma lista de Token.
  6. Os erros, como simbolos invalidos ou comentarios nao fechados, sao guardados em uma lista de mensagens.
  7. A classe Clone atualiza as tabelas de tokens e erros para o usuario estudar o resultado.

4. Classe Token.java

A classe Token e simples, mas fundamental. Ela guarda as informacoes produzidas pelo analisador lexico. Cada token possui lexema, classe, padrao, descricao e linha.

padraoRegra ou forma geral que descreve o token reconhecido.
descricaoExplicacao amigavel para exibir na interface e no relatorio exportado.

5. Classe Lexer.java

A classe Lexer contem o analisador lexico propriamente dito. Ela recebe uma string com o codigo inteiro e usa dois contadores principais: pos, que indica a posicao atual no texto, e linha, que indica a linha atual. Tambem possui duas listas: tokens e erros.

Palavras reservadas

O conjunto PALAVRAS_RESERVADAS lista termos da linguagem Pascal que nao devem ser classificados como nomes comuns. Exemplos: program, var, begin, end, if, while, function e procedure.

Fluxo principal da analise

while (pos < codigo.length()) {
    char atual = codigo.charAt(pos);

    if (Character.isWhitespace(atual)) consumirEspaco();
    else if (atual == '{') reconhecerComentarioChaves();
    else if (atual == '(' && proximoEh('*')) reconhecerComentarioParenteses();
    else if (atual == '/' && proximoEh('/')) reconhecerComentarioLinha();
    else if (Character.isLetter(atual)) reconhecerIdentificadorOuReservada();
    else if (Character.isDigit(atual)) reconhecerNumero();
    else if (atual == '\'') reconhecerString();
    else reconhecerSimbolo();
}

Metodos de Lexer

MetodoO que fazComo foi implementado
Lexer(String codigo) Recebe o codigo que sera analisado. Guarda a string no atributo codigo. A analise real so comeca em analisar().
analisar() Executa a analise lexica completa. Limpa listas anteriores, reinicia pos e linha, percorre todo o texto e chama metodos especificos conforme o caractere atual. No fim adiciona o token EOF.
consumirEspaco() Ignora espacos, tabs e quebras de linha. Se encontrar \n, incrementa o contador de linha. Depois avanca uma posicao.
reconhecerIdentificadorOuReservada() Reconhece palavras e nomes definidos pelo programador. Monta um texto enquanto houver letra, digito ou sublinhado. Depois compara a versao minuscula com PALAVRAS_RESERVADAS.
reconhecerNumero() Reconhece numeros inteiros e reais. Primeiro consome digitos. Se encontrar ponto seguido de digito e nao for intervalo .., continua consumindo a parte decimal e classifica como real.
reconhecerString() Reconhece textos delimitados por aspas simples. Comeca na aspa inicial, acumula caracteres ate a aspa final e registra erro se chegar em quebra de linha ou fim do codigo sem fechar.
reconhecerComentarioChaves() Reconhece comentarios no formato { ... }. Consome caracteres ate encontrar }. Atualiza linha em quebras de linha. Se nao fechar, registra erro.
reconhecerComentarioParenteses() Reconhece comentarios no formato (* ... *). Avanca duas posicoes para pular (* , procura a sequencia *) e registra erro se nao encontrar.
reconhecerComentarioLinha() Reconhece comentarios iniciados por //. Consome caracteres ate a quebra de linha. Esse comentario termina automaticamente no fim da linha.
reconhecerSimbolo() Reconhece operadores e separadores. Usa switch para tratar +, -, *, /, =, <, >, :, ., virgula, ponto e virgula, parenteses e colchetes.
adicionarSimbolo() Adiciona token de simbolo simples. Cria um Token com linha atual e avanca pos em uma unidade.
adicionarSimboloDuplo() Adiciona token composto por dois caracteres. Cria um Token e avanca pos em duas unidades. Usado para <=, <>, >=, := e ...
proximoEh(char esperado) Verifica o proximo caractere sem avancar a leitura. Confere se existe posicao seguinte e se ela contem o caractere esperado.
getErros() Retorna a lista de erros encontrados. Devolve uma nova lista para evitar alteracao direta da lista interna.
isPalavraReservada(String palavra) Informa se uma palavra pertence ao conjunto reservado. Converte para minusculas e consulta o conjunto PALAVRAS_RESERVADAS.
getPalavrasReservadas() Retorna as palavras reservadas cadastradas. Usa Collections.unmodifiableSet para impedir alteracao externa do conjunto.

7. Como a implementacao foi pensada

A implementacao separa a logica da analise lexica da interface grafica. Essa divisao e importante porque Lexer pode ser estudado e testado sem depender do Swing, enquanto Clone fica responsavel por mostrar dados e receber a interacao do usuario.

O Lexer usa uma tecnica comum em compiladores simples: leitura sequencial com um indice. Em vez de tentar interpretar o programa completo, ele olha apenas o caractere atual e, quando necessario, olha o proximo caractere com proximoEh(). Isso permite reconhecer simbolos compostos como :=, <= e ...

A interface foi implementada com Swing porque ele ja oferece componentes prontos, como JFrame, JTextPane, JTable, JMenuBar, JScrollPane e JFileChooser. O codigo personaliza esses componentes para parecer uma IDE: editor com fonte monoespacada, numeracao de linhas, temas escuros, barra lateral e tabelas de resultados.

9. Pontos fortes e limitacoes

Pontos fortesLimitacoes
Codigo dividido em modelo, analisador e interface. A analise e apenas lexica; nao valida gramatica completa do Pascal.
Reconhece palavras reservadas, identificadores, numeros, strings, comentarios, operadores e separadores. Strings usam regra simples e nao tratam todos os escapes possiveis.
Mostra erros com linha e permite navegar ate a linha do erro. A sugestao por IA depende de Ollama local e do modelo instalado.
Interface facilita estudo visual dos tokens. O arquivo Clone.java concentra muitas responsabilidades e poderia ser dividido em classes menores.

10. Conclusao

O projeto e uma boa base didatica para entender a primeira fase de um compilador. A classe Lexer mostra como transformar texto bruto em tokens classificados. A classe Token organiza os dados de cada unidade reconhecida. A classe Clone transforma tudo isso em uma ferramenta visual, permitindo escrever codigo, executar a analise, ver resultados, filtrar informacoes, exportar relatorios e estudar erros.

Para evoluir o projeto, o proximo passo natural seria criar um analisador sintatico, responsavel por verificar se a ordem dos tokens forma comandos validos segundo a gramatica da linguagem.
